# SPEC-0005: Simulation Analysis Laboratory

## Overview

The Simulation Analysis Laboratory is an advanced toolset for interpreting combat simulation data. It transitions the dashboard from simple logging to a scientific analysis platform capable of detecting balance regressions, statistical significance, and combat swinginess.

## Requirements

### Requirement: Statistical Significance Analysis

The system <span className="rfc-keyword shall">SHALL</span> calculate the statistical significance of win rates using confidence intervals (e.g., 95% CI) to ensure results are not due to chance.

#### Scenario: High Confidence Result

- **WHEN** Team A wins 85 out of 100 simulations.
- **THEN** The UI <span className="rfc-keyword shall">SHALL</span> display a high-confidence indicator (p &lt; 0.05).

#### Scenario: Low Confidence Result

- **WHEN** Team A wins 52 out of 100 simulations.
- **THEN** The UI <span className="rfc-keyword shall">SHALL</span> display a "Statistically Insignificant" warning.

### Requirement: Delta/Comparison Analysis

The system <span className="rfc-keyword shall">SHALL</span> provide a comparison view between two simulation runs that highlights the absolute and relative differences in DPR (Damage Per Round) and survival rates.

#### Scenario: Subclass Comparison

- **WHEN** Comparing a Champion Fighter run vs a Battlemaster Fighter run.
- **THEN** The UI <span className="rfc-keyword shall">SHALL</span> display a "Delta" table showing % difference in DPR and win rate.

### Requirement: Combat Categorization

The system <span className="rfc-keyword should">SHOULD</span> categorize combat results into qualitative buckets (Stomp, Close, Slog, etc.) based on the number of rounds and final HP of the survivors.

#### Scenario: Slog Detection

- **WHEN** A combat lasts more than 10 rounds.
- **THEN** The system <span className="rfc-keyword shall">SHALL</span> label it as a "Slog".

### Requirement: Math Transparency (Luck Tracking)

The system <span className="rfc-keyword shall">SHALL</span> calculate a "Luck Rating" for each combatant, comparing their actual d20 roll average against the statistical expectation (10.5).

#### Scenario: Visualizing Luck
- **WHEN** A simulation is analyzed in the Lab.
- **THEN** The UI <span className="rfc-keyword shall">SHALL</span> display a "Luck Rating" (e.g., +1.2 above avg) for each team.

### Requirement: Roll Distribution Visualization

The system <span className="rfc-keyword shall">SHALL</span> provide a histogram visualization of all d20 rolls in a simulation run to verify dice engine fairness and identify "swingy" results.

#### Scenario: Dice Histogram
- **WHEN** the Roll Inspector is opened.
- **THEN** The system <span className="rfc-keyword shall">SHALL</span> display a chart showing the frequency of results from 1 to 20.
