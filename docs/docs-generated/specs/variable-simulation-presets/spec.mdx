# SPEC-0006: Variable Simulation Presets

## Overview

Simulation presets are currently static. To perform scientific analysis (e.g., "How many Goblins can a Level 5 Fighter handle?"), users must manually configure multiple custom simulations. This capability introduces "Variable Presets" that allow defining a range for a variable (like enemy count or level) and running a "Parameter Sweep" to visualize the trend.

## Requirements

### Requirement: Parameter Sweep Definition

The JSON preset schema <span className="rfc-keyword shall">SHALL</span> support variable ranges using a `variables` key.

#### Scenario: Enemy Count Scaling
- **WHEN** A preset defines `variables: { "goblin_count": [1, 2, 4, 8, 12] }`.
- **THEN** The API <span className="rfc-keyword shall">SHALL</span> generate and run a batch of 5 simulations, one for each value in the range.

### Requirement: Variable Substitution

The system <span className="rfc-keyword shall">SHALL</span> support placeholders in the `teams` configuration that reference defined variables.

#### Scenario: Placeholder Replacement
- **WHEN** a team member uses `{ "count": "{{goblin_count}}" }`.
- **THEN** The system <span className="rfc-keyword shall">SHALL</span> duplicate that member template the specified number of times.

### Requirement: Variable Expression Control (UI)

The UI <span className="rfc-keyword shall">SHALL</span> provide an interface for users to define parameter sweeps (variables) without editing JSON files.

#### Scenario: Defining a Sweep in UI
- **WHEN** using the Scenario Configurator.
- **THEN** The user <span className="rfc-keyword must">MUST</span> be able to add a variable (e.g., `goblin_count`) and a list of values (e.g., `[1, 2, 4]`).
- **THEN** The user <span className="rfc-keyword must">MUST</span> be able to reference that variable in team member counts or fields using `{{variable_name}}`.

### Requirement: Script/UI Parity

The system <span className="rfc-keyword shall">SHALL</span> ensure that any experiment run via a Ruby script can be represented as a Variable Preset in the UI.

#### Scenario: Subclass Comparison Sweep
- **WHEN** A variable `subclass` is defined as `["champion", "battlemaster"]`.
- **THEN** The UI <span className="rfc-keyword shall">SHALL</span> display a side-by-side comparison of the trends for both subclasses.
